# Аудит — 2026-05-14

## Изменения с прошлого аудита (2026-04-04)

### Безопасность
- **Auth-ошибки маппятся в обобщённые** в `AuthPage.tsx` (`mapAuthError`). Нет
  user enumeration.
- **`generate-document`** Edge Function теперь проверяет авторизацию: admin /
  модуль `sales_leads` / `responsible_id` заказа / партнёр-владелец. Возвращает 403
  иначе. Закрыта дыра: ранее любой авторизованный мог скачать счёт по чужому заказу.
- **Bucket `partner-attachments`** переведён в private. SELECT-политика разрешает
  чтение только admin / `sales_leads` / владельцу / назначенному менеджеру (через
  `partner_requests` и `partner_request_messages.attachment_urls`). DELETE —
  admin / автор загрузки.
- Подтверждены допущения: realtime через `postgres_changes` (RLS
  source-table), `integration_queue` admin-only.

### Новые модули
- **Расчёт ЗП (`/payroll`)** — сотрудники, операции, табель, отчёт A4-печать.
  Только admin.
- **`OperationsTab`** — CRUD операций производства со стоимостью часа работы.
- Справочник **`payroll_skill_coefficient`** (Junior/Middle/Senior/Master =
  0.8/1.0/1.3/1.6) — редактируется в админке. Коэффициент подставляется в табель
  автоматически по уровню сотрудника, можно переопределить.

### Багфиксы
- **`LeadDetailDialog`** — устранено зависание при открытии карточки лида. Причина:
  ранний `return null` стоял ДО `useQuery`, что нарушало порядок React-хуков и
  вешало приложение при повторных открытиях. Хуки теперь вызываются всегда,
  `.single()` заменён на `.maybeSingle()`.
- Карточка лида показывает: контакты, прогресс по статусам, привязанный расчёт,
  клиент, созданный заказ, заметки.
