## 🔐 Безопасность

### Аутентификация
- Email/password через Auth
- Профиль создаётся автоматически (trigger `handle_new_user`)
- Роль `user` назначается по умолчанию (trigger `handle_new_user_role`)

### RBAC (Role-Based Access Control)
- **Enum `app_role`:** `admin`, `user`
- **Таблица `user_roles`:** связь user_id ↔ role
- **Группы доступа:** `access_groups` → `group_permissions` → `user_group_assignments`
- **Модули:** `calculator`, `history`, `clients`, `docs`
- **Функция `has_module_access`:** админы всегда имеют доступ

### Аудит
- ✅ `npm audit` — 0 уязвимостей (high/critical)
- ✅ Линтер БД — 0 проблем
- ✅ DOMPurify для санитарной обработки HTML
- ✅ SECURITY DEFINER на функциях проверки ролей (предотвращение рекурсии RLS)
